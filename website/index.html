Install

Up and running in under a minute.

One installer, all three hosts. Local state lives in ~/.valor/.

bash 
# one-line install — clones to ~/.valor/repo, installs for all hosts
$ curl -fsSL https://raw.githubusercontent.com/yihanzhu/valor/main/install.sh | bash -s -- --clone

Installs Valor for Claude Code, Codex CLI, and Cursor in one go. Auto-checks for daily updates.

bash 
$ git clone https://github.com/yihanzhu/valor.git ~/.valor/repo
$ cd ~/.valor/repo
$ bash install.sh

Inspect the installer before running it. Same result as the one-liner, with the source on disk.

bash 
$ bash install.sh --target claude-code   # Claude Code only
$ bash install.sh --target codex         # Codex CLI only
$ bash install.sh --target cursor        # Cursor only

Only install Valor for the agent you actually use. Switch on or off later by re-running.

bash 
$ cd ~/.valor/repo && bash install.sh --upgrade

Manual upgrade. Valor also auto-applies minor and patch releases when you start a new session.

Features

Eight workflows. One running coaching loop.

Every command runs against your local evidence store. No dashboards, no telemetry.

Morning

Start the day with priorities, not inbox.

/valor-briefing pulls open Jira tickets, your PR queue, today's calendar, and yesterday's carry-forward — then proposes a focus order grounded in your target-level competencies.

See the spec →
claude-code — /valor-briefing 
$ /valor-briefing

Tickets (Jira)
  ● AUTH-412   Retry storm under load        In Progress
  ● AUTH-418   Session cache invalidation    In Review

Pull requests
  #418 auth: retry policy w/ jittered backoff
  #421 deps: bump httpx 0.27 → 0.28

Coaching nudge
  Add a one-line rollback note to #418. That's
  the L5 coordination signal your framework
  weights most this quarter.

In flow

Senior-level review, before you hit "Request changes".

/valor-pr-review walks architecture, blast radius, test gaps, and tone of the review itself. /valor-design-doc does the same for design — options, trade-offs, and what to explicitly punt.

PR review playbook →
cursor — /valor-pr-review 
$ /valor-pr-review #418

Architecture
  Retry policy lives in the wrong layer.
  Move from api.client → net.transport.
  Why: future RPCs need it too.

Test gaps
  ! No coverage for jitter bounds.
  ! Backoff cap untested above 30s.

Tone check
  Comment on line 84 reads sharper than
  intended — soften to a question.

✓ Logged as evidence: code_review · subject_matter

Evidence

Your work, queryable. Locally.

Every command writes structured evidence to ~/.valor/evidence.sqlite. Export it for weekly reflection, 1:1 prep, or self-review — no cloud, no scraping.

Architecture →

Wrap-up

End the day with carry-forward, not loose ends.

/valor-wrapup reads the day's conversation, git history, and evidence — then writes a handoff file you (or tomorrow-you) can pick up cold.
